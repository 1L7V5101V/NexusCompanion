可以把“漂流链路 Drift”理解成：**Agent 没有东西需要主动推送时，不是原地空转，而是利用这段空闲时间做一些后台自我维护或探索任务**。

比如：

```text
没有 GitHub 告警
没有日历提醒
没有 RSS 内容值得推
用户也没发消息
```

这时主动链路就会说：

> 既然现在没东西要通知用户，那我能不能做点有价值的后台工作？

这就是 Drift。

**1. Drift 在整个系统里的位置**
项目有三条链路：

```text
被动链路：用户问，Agent 答
主动链路：外部有事，Agent 主动推
漂流链路：没事可推，Agent 自己做后台任务
```

Drift 不是用户直接触发的，而是由主动链路触发。

在 [agent/core/proactive_turn.py](<D:\.Projects\NexusCompanion\agent\core\proactive_turn.py>) 里能看到触发条件：

```python
if not gw_result.alerts and not gw_result.content_meta and not ctx.context_as_fallback_open:
    if self._drift_pipeline is not None and self._cfg.drift_enabled:
        entered_drift = await self._drift_pipeline.run(ctx, self._llm_fn)
```

翻译成人话：

```text
如果没有 alert
也没有 content
context fallback 也没开
并且 drift 已启用
那就进入 Drift
```

所以 Drift 的定位很清楚：

> 它是主动推送链路没内容时的备用后台工作链路。

**2. Drift 做什么**
Drift 做的不是“随便聊天”，而是执行一个个任务脚本。

这些任务由 `SKILL.md` 定义，比如：

```text
整理长期记忆
审计用户画像
生成日报草稿
检查最近上下文是否需要归档
补全某个资料库
创建新的 drift skill
管理表情包/素材
```

它很像一个后台实习生：没人找它时，它会翻自己的任务清单，找一件适合现在做的事，做一点进度，然后记录下来。

**3. Drift 的核心代码入口**
核心类在 [agent/core/drift_turn.py](<D:\.Projects\NexusCompanion\agent\core\drift_turn.py:78>)：

```python

```

主入口是：

```python
async def run(self, ctx, llm_fn) -> bool:
```

它分四步：

```text
Scan    扫描可用 skills
Prepare 准备工具和 prompt
Execute 让 LLM 按工具循环执行任务
Finish  记录退出状态
```

代码里也写得很直白：

```python

tools, messages, mounted_tool_names = self._prepare(ctx, skills)
await self._execute_loop(ctx, llm_fn, tools, messages, mounted_tool_names)
self._finish(ctx)
```

**4. 第一步：Scan，扫描可做的任务**
Drift 的任务来自 `drift/skills/*/SKILL.md`，也可以包含内置 skill。

扫描逻辑在 [proactive_v2/drift_state.py](<D:\.Projects\NexusCompanion\proactive_v2\drift_state.py:92>)：

```python
def scan_skills(self) -> list[SkillMeta]:
```

每个 skill 会被解析成：

```python
class SkillMeta:
    name
    description
    last_run_at
    run_count
    status
    next
    requires_mcp
    builtin
```

也就是说，Drift 会知道：

```text
这个任务叫什么
它上次什么时候跑过
跑过几次
下一步是什么
是否需要某个 MCP server
是不是内置任务
```

然后在 [agent/core/drift_turn.py](<D:\.Projects\NexusCompanion\agent\core\drift_turn.py:113>) 里过滤：

```python
skills = self._store.scan_skills()

connected_servers = shared.get_mcp_server_names() if shared else set()
skills = [
    s for s in skills
    if not s.requires_mcp or set(s.requires_mcp) <= connected_servers
]
```

意思是：

> 如果某个 Drift 任务依赖某个 MCP server，但当前没连上，那这个任务先不跑。

比如一个 skill 需要 `github` MCP，但 GitHub server 没连上，就跳过。

**6. Drift 工作区是什么**
Drift 有自己的工作区：

```text
workspace/proactive.db 所在目录下的 drift/
```

代码里是：

```python
drift_dir = Path(self._deps.state_store.workspace_dir) / "drift"
```

这个目录下面一般会有：

```text
drift/
  skills/
    xxx/
      SKILL.md
      工作文件...
  drift.json
```

`SKILL.md` 是任务说明，`drift.json` 是运行记录。

这点很重要：Drift 不是完全靠模型临场发挥，而是有文件系统状态和任务说明来约束它。

**7. 第三步：Execute，工具调用循环**
Drift 真正运行时，不是让模型直接输出一段话，而是强制它调用工具。

在 [agent/core/drift_turn.py](<D:\.Projects\NexusCompanion\agent\core\drift_turn.py:175>)：

```python
while steps < self._max_steps and not ctx.drift_finished:
    tool_choice = "required"
    tool_call = await llm_fn(messages, schemas, tool_choice, disable_thinking=True)
```

关键是：

```python
tool_choice = "required"
```

意思是模型每一步都必须选择一个工具调用。

比如它可能这样走：

```text
read_file("skills/memory-audit/SKILL.md")
 -> read_file("skills/memory-audit/state.md")
 -> recall_memory("最近用户画像")
 -> write_file("skills/memory-audit/report.md")
 -> finish_drift(...)
```

每执行一个工具，结果会被追加回 messages：

```python
self._append_tool_messages(...)
```

这样模型下一步能看到上一步工具结果，继续做任务。

**8. Drift 为什么不会一直跑飞**
这里有几层防跑飞设计。

第一，最大步数限制：

```python
while steps < self._max_steps
```

默认来自配置：

```python
max_steps=getattr(self._deps.cfg, "drift_max_steps", 20)
```

第二，必须调用 `finish_drift` 才算正常结束。

`finish_drift` 在 [proactive_v2/drift_tools.py](<D:\.Projects\NexusCompanion\proactive_v2\drift_tools.py:111>)：

```python
class FinishDriftTool(Tool):
```

它要求必须提供：

```text
skill_used
one_line
next
message_result
```

也就是：

```text
这次用了哪个 skill
这轮做了什么
下次该做什么
有没有给用户发消息
```

第三，`message_push` 最多一次。

在 [proactive_v2/drift_tools.py](<D:\.Projects\NexusCompanion\proactive_v2\drift_tools.py:82>)：

```python
if self._ctx.drift_message_sent:
    return {"error": "message_push already used in this drift run"}
```

第四，一旦发过消息，后续工具会被限制。

在 [agent/core/drift_turn.py](<D:\.Projects\NexusCompanion\agent\core\drift_turn.py:197>)：

```python
if ctx.drift_message_sent:
    allowed_after_send = {"write_file", "edit_file", "finish_drift"}
    schemas = [
        s for s in schemas
        if s["function"]["name"] in allowed_after_send
    ]
```

意思是：

> Drift 一旦给用户发了消息，就不能继续 web_search、shell、recall_memory 乱跑了，只能写文件记录状态，然后 finish_drift 收尾。

这个设计很关键，防止 Agent 发完消息后继续在后台做不可控动作。

**9. 第四步：Finish，保存状态**
`finish_drift` 会写入 `drift.json`。

在 [proactive_v2/drift_state.py](<D:\.Projects\NexusCompanion\proactive_v2\drift_state.py:168>)：

```python

```

它会记录：


这些记录下次 Drift 会重新注入 prompt。

所以 Drift 有“连续工作”的能力：

```text
这次做一点
记录下一步
下次空闲时接着做
```

这就是它不像普通一次性任务的地方。

也就是说：


这样 Drift 发出去的消息也能被记录为 proactive 消息，并参与去重、审计、trace。

**11. 给初学者的完整例子**
假设现在主动链路没内容可推，Drift 被触发。

它可能这样运行：

```text
1. Proactive tick 发现：
   alert = 0
   content = 0
   context fallback = false

2. 进入 DriftTurnPipeline.run()

3. 扫描 drift/skills：
   - memory-audit
   - daily-draft
   - meme-manage

4. 选择一个 skill，比如 daily-draft

5. 读取：
   skills/daily-draft/SKILL.md

6. 按 SKILL.md 说明继续：
   - 读取昨天的上下文
   - 整理草稿
   - 写入 draft.md

7. 如果没必要打扰用户：
   finish_drift(
     skill_used="daily-draft",
     one_line="整理了日报草稿第一版",
     next="下次补充今天的完成事项",
     message_result="silent"
   )

8. 如果发现真的值得告诉用户：
   message_push("我顺手整理了今天的日报草稿...")
   write_file(...)
   finish_drift(..., message_result="sent")
```

**12. 和主动链路的区别**
主动链路 Proactive 主要问：

```text
外部新信息值不值得推给用户？
```

Drift 主要问：

```text
没有外部新信息时，我能不能自己推进一点长期任务？
```

所以：

```text
Proactive = 筛选信息，决定推不推
Drift = 执行任务，推进后台工作
```

**13. 面试讲法**
你可以这样讲：

> Drift 是主动链路无 alert、无 content、且 context fallback 未开启时触发的空闲任务链路。它会扫描 drift/skills 下的 SKILL.md，过滤掉 MCP 依赖不满足的任务，然后构造 Drift 专用工具注册表和 runtime context，让 LLM 以 tool_choice=required 的方式进入工具调用循环。执行过程中可以读写 Drift 工作区、检索记忆、挂载 MCP 工具，也可以最多 message_push 一次。发消息后工具集合会收缩到 write/edit/finish，避免继续不可控探索。最后必须通过 finish_drift 写入 skill_used、one_line、next 和 message_result，使任务能跨 tick 持续推进。

短版：

> Drift 让 Agent 在没有内容可推时利用空闲时间执行可编程后台任务。它用 SKILL.md 定义任务，用工具调用循环执行，用 finish_drift 强制记录进度，用 message_push 限制和统一发送出口保证安全。
