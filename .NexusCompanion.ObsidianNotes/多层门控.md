主动链路的“门控”主要在 agent/core/proactive_turn.py 里实现，核心入口是 [[ProactiveTurnPipeline.run]]()。

---
“门控”就是 **gate**，可以理解成：**在真正执行某个动作前，加一层判断开关**。

在这个项目里，门控的作用是：

> Agent 不是想推送就推送、想调用工具就调用工具，而是先经过一系列规则判断，只有通过了才继续执行。

比如主动推送前会问：

```text
现在用户适合被打扰吗？
刚刚是不是已经推送过？
今天是不是推送太多次了？
被动对话链路是不是正在忙？
这条内容是不是重复？
这条内容价值够不够高？
```

这些判断就是门控。

举个生活例子：

```text
你想给朋友发消息
 -> 他是不是在睡觉？
 -> 刚刚是不是已经发过？
 -> 这件事是否真的重要？
 -> 他最近是不是很忙？
 -> 都通过了，才发
```

放到 Agent 里就是：

```text
候选推送内容
 -> 冷却门控
 -> busy 门控
 -> presence 门控
 -> quota 门控
 -> 去重门控
 -> LLM 价值判断
 -> 发送
```

几个常见门控含义：

- **冷却门控**：刚发过消息，短时间内不再发。
- **busy 门控**：用户正在和 Agent 对话时，不插入主动推送。
- **presence 门控**：根据用户活跃状态判断是否适合打扰。
- **quota 门控**：限制每天主动推送次数。
- **dedupe 门控**：避免重复推送同一件事。
- **LLM 门控**：让模型判断内容是否真的值得说。

面试里可以这样讲：

> 门控是主动链路里的决策过滤层。每次 Agent 想主动行动前，都要经过冷却、忙碌状态、可打扰度、配额、去重和 LLM 价值判断等 gate，确保主动能力不会变成无意义打扰。
